# Anvora Workspace Rule

Activation: Always On

## Project

Anvora is a premium AI Operating System for founders, traders, and businesses.

The codebase must prioritize:

- Clean architecture
- Reusable components
- Strong TypeScript typing
- Accessibility
- Responsive layouts
- Production-ready code
- Maintainability over shortcuts

## Frontend Stack

- React 19
- TypeScript
- Vite
- Tailwind CSS v4
- shadcn/ui
- React Router
- Lucide React

## UI Principles

- Apple-quality visual polish
- Dark executive theme
- Glass effects only where they improve hierarchy
- Consistent spacing (8px grid)
- Rounded corners
- Minimal animations
- Professional appearance over flashy effects

## Folder Structure

- src/components → reusable UI
- src/features → feature modules
- src/layouts → application layouts
- src/app → routing and providers
- src/lib → shared utilities

## Coding Rules

- Prefer functional React components.
- Use TypeScript interfaces for public props.
- Avoid duplicated logic.
- Keep components focused and reusable.
- Do not introduce unnecessary dependencies.
- Follow existing project structure before creating new folders.

## Output Expectations

When generating code:

1. Explain the architecture briefly.
2. Create all required files.
3. Keep code production-ready.
4. Avoid placeholder implementations unless requested.
